# WriteEmail & 如何写邮件
### 邮件书写的六要素
* 标题
* 收件人/上款
* 时间点
* 抄送
* 突出重点
* 当面或电话沟通，邮件中再次确认
###　邮件标题
* 邮件一定要有标题，但也不要太长，一般不超过16字；
* 标题要表达本Email的主要意图
* 标题可以加称呼，以引起收件人的注意；
* 举例：
-- “提请OP注意，JXQ IDC一起的服务器即将搬迁” 是一个好的标题。
-- 而要进行同样的事情，“JXQ IDC一期的服务器列表”就是一个效果不好的标题。
### 邮件收件人
* 考虑清楚邮件的收件人，区分MailTo和CC；
* 明白写好上款，以便对方明确意识到“自己有细看这个邮件的责任”，也让不需要仔细看该邮件的收件人一目了然；
* 上款宜直接称呼对方的名字，如果有多人，宜逐个都称呼到；
--如“Robin、旭阳、William，各位好，现有关于XXX的方案需要你们审核...” 
* 如果人数比较多、无法逐个称呼名字，也要设法归纳出一个上款；
-- 如“程序中用到Link域的同事，请注意：Spider计划调整该域的格式...”
-- 如“HR负责员工关怀的同事，你们好，有件事情需要请问一下...”
* 一个Email中给多个人分别交代事情，应该分成多段，每段分别写明上款。
### 时间点
* 给对方提出需求的，须写明希望的时间点，风格应该明确但不生硬；
-- 如“这件工作需要在15日下班前完成，如果你觉得这个时间点有问题，请今天回信或来电话我们商量”
-- 这样，即明确了时间点，又给了对方余地，对方既感觉比较舒服，而如果他不反驳就意味着他接受了这个时间点；
* 对方提出需求的：如果他没有写时间点、紧急程度、自己应该在沟通了紧急程度和重要性之后，回信中给出完成时间点；
* 跟踪BUG/Case的回复邮件：必须写明问题、分析、结论、计划、时间点。
### 抄送
* 跨部门、跨组的合作事宜，需要朝东双方直接经理；
* 在拿不准是否应该抄送经理时，一律抄送；
* 同理，如果其他部门的人给你写的Email没有抄送经理，你可以不予安排而直接打回，请他再发送并抄送给双方的经理。
### 突出重点
* Email中安排的时间点，须通过加粗、飘红、飘蓝方式突出显示；
* 任务安排中的人名要突出显示，以便引起足够的注意；
* 多段中提到不同的收件人时，每个收件人要突出显示；
* 另一方面，慎重感叹号和颜色，不要乱标重点。
### 当面、电话沟通之后，邮件中再次确认
* 如果事先已当面或电话沟通过，也要在邮件中写明意图、结论、分工、时间点等等，不能因为“都说过了”就省事不写了；
-- 邮件是一种成文的记录，通过邮件确认可以避免歧义、并予以强调 
* 同样，如果已经当面或电话沟通过，还一定要在邮件中写清楚我们“已经直接沟通过了”；
-- 因为，邮件抄送人中还有别人，包括经理，他们都可能怀疑“这件事光靠邮件能说得清楚吗？”
-- 这时你应该清晰地告诉大家“我们已经当面讨论过了”，这样大家就都放心了 

### 进阶
* 以上六点是email的基本要求，与每个人的语文功底没什么关系，只要沟通意识、时间意识有了，就能做好。 
* 更进一步，是需要注意语法，避免错字病句。这与语文水平有点关系，然而，如果写完email后自己能先看一遍，就可以有效的减少错别字和病句。 
* 更好的email，是要注意布局、段落合理、适当修辞、情态生动。相信很多人实际上也有实力达到这个水平。 
* 如果某个email的收件人很多，那么写完后不要着急发出去，可以先给自己的经理看一遍，这样容易保证质量，既避免因email写得不好耽误很多人的时间、也能使得自己在很多人面前表现得体、树立形象。
